## ⚠️ Nouvelle Erreur Découverte

### ERREUR 5 : String Interpolation avec Helpers HTML (switch_domain_banner)

**Description :** J'ai converti la ligne HAML suivante incorrectement :
```haml
= "#{helpers.link_to "demarche.numerique.gouv.fr", new_host_url}."
```

**Ma conversion ERB (INCORRECTE) :**
```erb
<%= "#{helpers.link_to("demarche.numerique.gouv.fr", new_host_url)}." %>
```

**Problème :**
- `link_to` retourne un `SafeBuffer` (HTML safe)
- L'interpoler dans une string `"#{...}"` le convertit en String normale
- `<%= %>` échappe alors le HTML → le lien est affiché comme texte, pas comme HTML

**Impact :** 2 tests échoués en CI (switch_domain_banner_component_spec)

**Solution (CORRECTE) :**
```erb
<%= helpers.link_to("demarche.numerique.gouv.fr", new_host_url) %>.
```
Sortir le `.` de l'interpolation et le mettre en texte brut.

**Root Cause :** Je n'ai pas compris que HAML gère différemment les SafeBuffer dans l'interpolation

**Temps perdu :** ~5 min (correction + amend)

**Gravité :** 🟡 MOYENNE - Pattern peu courant, difficile à anticiper sans tests
